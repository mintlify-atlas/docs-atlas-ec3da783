---
title: "Change MySQL Root Password"
description: "Reset or change the MySQL/MariaDB root password using init file or safe mode methods."
---

If you need to reset the MySQL root password — for example, after a fresh installation or if you've lost access — you can use one of the two methods below.

<Warning>
  Changing the root password will affect all services that authenticate with it, including Frappe sites. Update your bench configuration after changing the password.
</Warning>

## Method 1: Using an init file (recommended)

This method uses a MySQL init file to run an `ALTER USER` statement on startup.

<Steps>
  <Step title="Stop the MySQL service">
    ```bash
    sudo systemctl stop mysql
    killall mysqld
    ```
  </Step>

  <Step title="Create the init file">
    Replace `[username]` with your Linux username:

    ```bash
    sudo nano /home/[username]/mysql-init
    ```

    Add the following line to the file, replacing `NewPassword` with your desired password:

    ```sql
    ALTER USER 'root'@'localhost' IDENTIFIED BY 'NewPassword';
    ```

    Save and exit with `Ctrl+X`, `Y`, `Enter`.
  </Step>

  <Step title="Start MySQL with the init file">
    ```bash
    sudo mysqld --init-file=/home/[username]/mysql-init &
    ```
  </Step>

  <Step title="Start the MySQL service normally">
    ```bash
    sudo service mysql start
    ```
  </Step>
</Steps>

## Method 2: Using safe mode

This method starts MySQL without authentication to allow direct password updates.

<Warning>
  Safe mode disables authentication entirely while running. Do not leave MySQL running in this state on a production server.
</Warning>

<Steps>
  <Step title="Start MySQL in safe mode">
    ```bash
    sudo mysqld_safe --skip-grant-tables --skip-networking &
    ```
  </Step>

  <Step title="Connect to MySQL as root">
    ```bash
    sudo mysql -u root
    ```
  </Step>

  <Step title="Update the root password">
    Run the following SQL commands, replacing `new_password` with your desired password:

    ```sql
    USE mysql;
    UPDATE user SET authentication_string=PASSWORD('new_password') WHERE User='root';
    FLUSH PRIVILEGES;
    ```
  </Step>

  <Step title="Exit MySQL">
    ```sql
    EXIT;
    ```
  </Step>

  <Step title="Kill the safe mode process">
    ```bash
    sudo killall mysqld_safe
    ```
  </Step>

  <Step title="Start the MySQL service normally">
    ```bash
    sudo systemctl start mysql
    ```
  </Step>
</Steps>

<Note>
  After changing the root password, update any Frappe site configurations that reference the old password by editing the `site_config.json` for each site or re-running `bench setup`.
</Note>
