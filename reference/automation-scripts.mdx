---
title: "Automation scripts"
description: "Shell scripts that automate the full Frappe and ERPNext installation and configuration workflow."
---

These scripts cover every phase of a Frappe installation — from installing system packages through configuring a production environment. Run them in order for a clean setup.

<Warning>
  Review each script before running it. Scripts that invoke `sudo` will modify system-level configuration files.
</Warning>

## Making scripts executable

Download or copy the scripts to your server, then make them executable:

```bash
chmod +x install_frappe_packages.sh
chmod +x init_frappe_bench.sh
chmod +x create_frappe_site.sh
chmod +x config_supervisor.sh
chmod +x config_nginx.sh
chmod +x config_redis.sh
chmod +x config_production_site.sh
```

## Scripts

<AccordionGroup>
  <Accordion title="1. install_frappe_packages.sh — Install system dependencies">
    Installs all required system packages, MariaDB, and configures the database for UTF-8 unicode support. Also sets up the Python virtual environment and installs `frappe-bench`.

    ```bash install_frappe_packages.sh
    #!/bin/bash
    sudo apt-get update
    sudo apt-get install -y git python3-dev python3-setuptools python3-pip python3.10-venv virtualenv \
      libmysqlclient-dev redis-server nodejs yarn xvfb libfontconfig wkhtmltopdf \
      software-properties-common dirmngr apt-transport-https
    sudo apt update
    sudo apt install mariadb-server

    # Configure MariaDB for UTF-8 unicode (automated)
    CONFIG_FILE="/etc/mysql/my.cnf"

    # Add to [mysqld] section
    sudo tee -a $CONFIG_FILE > /dev/null <<EOF
    [mysqld]
    character-set-client-handshake = FALSE
    character-set-server = utf8mb4
    collation-server = utf8mb4_unicode_ci
    EOF

    # Add to [mysql] section
    sudo tee -a $CONFIG_FILE > /dev/null <<EOF
    [mysql]
    default-character-set = utf8mb4
    EOF

    # Set up Python virtual environment
    python3 -m venv my_frappe_env
    source my_frappe_env/bin/activate
    pip3 install frappe-bench

    sudo apt install supervisor
    sudo apt install nginx
    ```

    **Usage:**
    ```bash
    ./install_frappe_packages.sh
    ```
  </Accordion>

  <Accordion title="2. init_frappe_bench.sh — Initialise the bench directory">
    Creates the `frappe-bench` directory, checks out the specified Frappe branch, and starts the development server. Defaults to `version-15` if no branch is supplied.

    ```bash init_frappe_bench.sh
    #!/bin/bash
    FRAPPE_BRANCH="${1:-version-15}"
    bench init frappe-bench --frappe-branch ${FRAPPE_BRANCH}
    cd frappe-bench
    source env/bin/activate
    bench start
    ```

    **Usage:**
    ```bash
    ./init_frappe_bench.sh [frappe_branch]
    ```

    | Argument | Default | Description |
    |---|---|---|
    | `frappe_branch` | `version-15` | The Frappe release branch to check out |
  </Accordion>

  <Accordion title="3. create_frappe_site.sh — Create a new site">
    Creates a new Frappe site and connects it to the specified database. All arguments are optional — the script will prompt for any that are missing.

    ```bash create_frappe_site.sh
    #!/bin/bash
    SITE_NAME="$1"
    DB_TYPE="${2:-mariadb}"
    DB_HOST="${3:-localhost}"
    DB_PORT="${4:-3306}"
    DB_USERNAME="${5:-root}"
    DB_PASSWORD="${6}"
    ADMIN_PASSWORD="${7}"

    # Prompt for required values if not provided
    if [ -z "$SITE_NAME" ]; then
      read -p "Enter site name: " SITE_NAME
    fi
    if [ -z "$DB_PASSWORD" ]; then
      read -sp "Enter DB root password: " DB_PASSWORD; echo
    fi
    if [ -z "$ADMIN_PASSWORD" ]; then
      read -sp "Enter admin password: " ADMIN_PASSWORD; echo
    fi

    bench new-site \
      --db-name ${SITE_NAME} \
      --db-password ${DB_PASSWORD} \
      --db-type ${DB_TYPE} \
      --db-host ${DB_HOST} \
      --db-port ${DB_PORT} \
      --mariadb-root-username ${DB_USERNAME} \
      --mariadb-root-password ${DB_PASSWORD} \
      --no-mariadb-socket \
      --admin-password ${ADMIN_PASSWORD}
    ```

    **Usage:**
    ```bash
    ./create_frappe_site.sh [site_name] [db_type] [db_host] [db_port] [db_username] [db_password] [admin_password]
    ```

    | Argument | Default | Description |
    |---|---|---|
    | `site_name` | _(prompted)_ | Hostname for the new site |
    | `db_type` | `mariadb` | Database engine |
    | `db_host` | `localhost` | Database host |
    | `db_port` | `3306` | Database port |
    | `db_username` | `root` | Database root username |
    | `db_password` | _(prompted)_ | Database root password |
    | `admin_password` | _(prompted)_ | Frappe administrator password |
  </Accordion>

  <Accordion title="4. config_supervisor.sh — Configure Supervisor">
    Generates the Supervisor configuration for the bench and links it into `/etc/supervisor/conf.d/`.

    ```bash config_supervisor.sh
    #!/bin/bash
    SITE_NAME="$1"
    bench --site ${SITE_NAME} setup supervisor
    sudo ln -s $(pwd)/config/supervisor.conf /etc/supervisor/conf.d/frappe-bench.conf
    ```

    **Usage:**
    ```bash
    ./config_supervisor.sh [site_name]
    ```
  </Accordion>

  <Accordion title="5. config_nginx.sh — Configure Nginx">
    Generates the Nginx configuration for the bench and links it into `/etc/nginx/conf.d/`.

    ```bash config_nginx.sh
    #!/bin/bash
    SITE_NAME="$1"
    bench --site ${SITE_NAME} setup nginx
    sudo ln -s $(pwd)/config/nginx.conf /etc/nginx/conf.d/frappe-bench.conf
    ```

    **Usage:**
    ```bash
    ./config_nginx.sh [site_name]
    ```
  </Accordion>

  <Accordion title="6. config_redis.sh — Configure Redis">
    Generates the Redis configuration files required by the bench.

    ```bash config_redis.sh
    #!/bin/bash
    SITE_NAME="$1"
    bench --site ${SITE_NAME} setup redis
    ```

    **Usage:**
    ```bash
    ./config_redis.sh [site_name]
    ```

    <Note>
      To further tune Redis settings, edit `/etc/redis/redis.conf` after running this script.
    </Note>
  </Accordion>

  <Accordion title="7. config_production_site.sh — Final production configuration">
    Runs `bench setup production`, which ties together Supervisor, Nginx, and Redis for a production-ready deployment.

    ```bash config_production_site.sh
    #!/bin/bash
    SITE_NAME="$1"
    FRAPPE_USER="$2"
    sudo bench --site ${SITE_NAME} setup production ${FRAPPE_USER}
    ```

    **Usage:**
    ```bash
    ./config_production_site.sh [site_name] [frappe_user]
    ```

    | Argument | Description |
    |---|---|
    | `site_name` | The site to configure for production |
    | `frappe_user` | The system user that owns the bench directory |
  </Accordion>
</AccordionGroup>

  <Step title="Initialise the bench">


  <Step title="Create a site">
    ```bash
    ./create_frappe_site.sh [site_name] [db_type] [db_host] [db_port] [db_username] [db_password] [admin_password]
    ```
  </Step>
  <Step title="Configure Supervisor">
    ```bash
    ./config_supervisor.sh [site_name]
    ```
  </Step>
  <Step title="Configure Nginx">
    ```bash
    ./config_nginx.sh [site_name]
    ```
  </Step>
  <Step title="Configure Redis">
    ```bash
    ./config_redis.sh [site_name]
    ```
  </Step>
  <Step title="Finalise production setup">
    ```bash
    ./config_production_site.sh [site_name] [frappe_user]
    ```
  </Step>
</Steps>
